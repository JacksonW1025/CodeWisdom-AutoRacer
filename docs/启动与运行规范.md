根目录文档只维护系统约束、阶段目标、接口协议和验证边界。

阶段 1-4 的验收契约、追踪表、PASS/FAIL 规则和人工边界以根目录
`../../docs/阶段路线图.md` 和 `../../docs/开发流程与验证规范.md` 为准。本文只记录上位机可执行入口和 topic/launch 检查；测试结果必须回填根目录阶段验收追踪表。

`ackermann_chassis_bridge` 配置文件参数：

| 参数 | 当前值 |
| --- | --- |

是霍尔反馈超速保护阈值，不是自动控制命令限速，也不是阶段 1/2 测试速度目标。

### 启动前检查

```bash
ros2 pkg prefix turn_on_autoracer_robot
ros2 pkg executables turn_on_autoracer_robot
test -e /dev/ttyACM0
python3 tools/acceptance/stage1_acceptance_check.py --mode offline
```

必须确认：

- STM32 固件已烧录。
- UART4 线缆、波特率、共地和电平已确认。
- RC 接管、急停、刹车和断电路径可用。
- `counts_per_meter` 有实测值，或本次只做串口/telemetry bringup。
- `/imu/data_raw` 已由 `hipnuc_imu` 发布，或本次不验收 `/odom`。
- `/imu/data` 已由 `imu_filter_madgwick` 发布，或本次不验收 `/odom`。

`counts_per_meter` 标定：

```text
counts_per_meter = abs(sum(hall_delta_count)) / measured_distance_m
```

记录要求：

- 单次标定距离不小于 `2.0 m`。
- 同一方向至少执行 `3` 次。
- 保存每次 `measured_distance_m`、`sum(hall_delta_count)`、单次结果和均值。
- 最大值和最小值相对均值偏差不超过 `10%`。
- 阶段 1 当前采用 `counts_per_meter=13.545`，来源为霍尔 `10` 计数/圈和轮径 `0.235 m`，并经三次大于 `2 m` 的直线采集确认。
- 无标定记录时不记录 `/wheel_odom` 或 `/odom` PASS。

### 启动后检查

```bash
ros2 node list
ros2 topic list
ros2 topic echo /chassis_state --once
ros2 topic hz /chassis_state
ros2 topic hz /wheel_odom
ros2 topic hz /odom
ros2 run tf2_ros tf2_echo odom base_footprint
python3 tools/acceptance/stage1_acceptance_check.py --mode live
python3 tools/acceptance/stage1_acceptance_check.py --mode live --require-odom
```

必须检查：

- `/chassis_state` 有数据。
- `/wheel_odom` 在 `counts_per_meter>0` 且 telemetry 有效时发布。
- `/odom` 在 `use_ekf:=true` 且 `/imu/data`、`/wheel_odom` 有效时发布。
- 通信超时后 STM32 进入停车状态。
- 非零运动命令测试必须满足架空或安全场地条件。

验收工具边界：

- `stage1_acceptance_check.py --mode offline` 只检查协议帧、telemetry 解析、`counts_per_meter` 门禁、launch 契约和串口异常恢复契约，不打开串口、不发布运动命令。
- `stage1_acceptance_check.py --mode live` 只检查已运行 ROS topic 和类型，不发布运动命令。
- `--require-odom` 用于 `/imu/data`、`/wheel_odom` 和 EKF `/odom` 均预期存在的检查场景。
- 工具输出必须回填测试记录；失败项不得人工改成 PASS。

标准非零运动命令只用于架空或低速安全场地：

```bash
timeout 2s ros2 topic pub --rate 10 /ackermann_cmd autoracer_interfaces/msg/AckermannChassisCommand "{header: {frame_id: ''}, speed_mps: 0.10, steering_angle_rad: 0.0, enable: true, brake: false, clear_fault: false, emergency_stop: false}"
timeout 2s ros2 topic pub --rate 10 /ackermann_cmd autoracer_interfaces/msg/AckermannChassisCommand "{header: {frame_id: ''}, speed_mps: 0.10, steering_angle_rad: 0.15, enable: true, brake: false, clear_fault: false, emergency_stop: false}"
timeout 2s ros2 topic pub --rate 10 /ackermann_cmd autoracer_interfaces/msg/AckermannChassisCommand "{header: {frame_id: ''}, speed_mps: 0.10, steering_angle_rad: -0.15, enable: true, brake: false, clear_fault: false, emergency_stop: false}"
ros2 topic pub --once /ackermann_cmd autoracer_interfaces/msg/AckermannChassisCommand "{header: {frame_id: ''}, speed_mps: 0.0, steering_angle_rad: 0.0, enable: true, brake: true, clear_fault: false, emergency_stop: false}"
ros2 topic pub --once /ackermann_cmd autoracer_interfaces/msg/AckermannChassisCommand "{header: {frame_id: ''}, speed_mps: 0.0, steering_angle_rad: 0.0, enable: false, brake: true, clear_fault: false, emergency_stop: true}"
```

记录要求：

- `straight_low_speed`：直行命令，`actual_speed_mps` 和 `/wheel_odom` 方向正确。
- `left_low_speed`：左转命令，`steering_angle_rad` 为正。
- `right_low_speed`：右转命令，`steering_angle_rad` 为负。
- `stop_command`：停车命令，速度回零且 `brake_active` 证据完整。
- `timeout_stop`：停止发布命令后记录 `command_timeout` 和速度回零。
- `estop_command`：上位机急停命令后记录 `estop_active` 和速度回零。

## 阶段 2：路径跟踪

### 当前状态

阶段 2 固定目标入口、test path publisher、point-goal path publisher、Pure Pursuit tracker 和 diagnostics publisher 已实现。

阶段 2 验收 case 固定为 `straight_2m`、`left_arc_r2m`、`right_arc_r2m`、`s_curve`、`stop_at_end`，并扩展 `goal_2m_2m` 用于验证无避障点目标跟踪。未实现对应 fixture、tracker、诊断或上机证据时，测试记录必须标为 `未实现`、`FAIL` 或 `BLOCKED`，不得记为通过。

阶段 2 实现归属：

- 新建 `autoracer_path_tracking` 包承载 test path publisher、Pure Pursuit tracker；tracker 同步发布 diagnostics。
- `autoracer_path_tracking/goal_path_publisher` 负责把当前 `/odom` 位姿到 `goal_x_m/goal_y_m` 的直线生成为 `/path_tracking/path`。
- 新建 `autoracer_bringup/stage2_path_tracking.launch.py` 作为固定目标入口。
- 新建 `autoracer_bringup/stage2_goal_tracking.launch.py` 作为 odom 点目标入口。
- `stage2_path_tracking.launch.py` include 阶段 1 Ackermann 底盘链路，并启动 `autoracer_path_tracking` 中的阶段 2 节点。
- 阶段 2 tracker 默认启用零速预使能门控：发出非零运动命令前，先持续发布 `speed_mps=0`、`enable=true`，直到 `/chassis_state` 显示 `auto_enabled=true`、`command_timeout=false`、`rc_override_active=false`、`estop_active=false`。
- tracker 算法固定为 Pure Pursuit。

当前已准备阶段 2 path fixture 和 fake-odom tracker 检查：

```bash
python3 tools/acceptance/stage2_path_fixture_check.py
python3 tools/acceptance/stage2_tracker_fake_odom_check.py
```

fixture 位置：

```text
tools/acceptance/fixtures/stage2_paths/
```

这些 fixture 是 `nav_msgs/Path` 语义的标准输入，用于 tracker 离线和实车验收。fixture 静态检查本身不发布
`/ackermann_cmd`，fake-odom 检查只证明 tracker 软件输出满足方向、限幅和到点停车契约，不代表实车阶段 2 PASS。

固定目标入口：

```bash
ros2 launch autoracer_bringup stage2_path_tracking.launch.py counts_per_meter:=<实测值>
```

该命令当前可运行；实车运行前必须满足阶段 1 安全链路、`counts_per_meter` 标定和低速可接管条件。
启动后 tracker 会先执行零速预使能。若底盘停留在空闲 RC passthrough 或 command timeout 状态，
tracker 必须保持零速，不得直接释放非零路径命令。

点目标入口示例，不带避障，只验证从当前 `/odom` 位姿到目标点的路径跟踪。当前 `max_steering_angle_rad=0.262` 时，`(2,2)` 对前进-only Ackermann 跟踪过紧；实车验收目标使用 `(3,2)`：

```bash
ros2 launch autoracer_bringup stage2_goal_tracking.launch.py counts_per_meter:=13.545 goal_x_m:=3.0 goal_y_m:=2.0 target_speed_mps:=0.18
```

现场明确批准更高速度时，可以同时覆盖 tracker 的速度上限，例如 `target_speed_mps:=1.00 max_target_speed_mps:=1.00`。

点目标入口默认 `lock_start_pose=true`，即用首次有效 `/odom` 生成到目标点的路径后保持固定，不随车辆运动反复重规划。若前视目标落到车后方且 `allow_reverse=false`，tracker 必须输出停车/刹车命令，不得继续发布前进速度。

稳定版阶段 4 参数约束：

| 项 | 验收约束 |
| --- | --- |
| planner | 使用 `nav2_smac_planner/SmacPlannerHybrid` |
| controller | 使用 `nav2_regulated_pure_pursuit_controller/RegulatedPurePursuitController` 作为阶段 4 主线 |
| `minimum_turning_radius` | 使用实车标定值；当前文档事实为 `2.24 m` |
| 4A 倒车 | Smac `motion_model_for_search=DUBIN` 或等价无倒车 cusp 配置；RPP `allow_reversing=false` |
| 4B 倒车 | 可使用 `REEDS_SHEPP`；仅在后方/侧方安全区和倒车限速证据齐全时开启 |
| 原地转向 | 必须禁用；例如 RPP `use_rotate_to_heading=false` |
| Nav2 Twist | remap/隔离为 `/nav2_cmd_vel` 和 `/safe_nav2_cmd_vel`，不得直接发到底盘 |
| adapter diagnostics | 固定为 `/twist_to_ackermann/diagnostics`，类型 `diagnostic_msgs/msg/DiagnosticArray` |

阶段 4 默认完成范围：

- 阶段 4A 是稳定版阶段 4 的默认 PASS 范围；除 `reverse_plan_controlled` 外，阶段 4 验收 case 必须有 PASS 或明确的安全 `BLOCKED` 证据。
- `reverse_plan_controlled` 是阶段 4B 扩展项；阶段 4A 通过时可记录为 `未实现/4B 后续`，不得伪记为 PASS。
- 阶段 4B 只有在倒车路径、倒车限速、后方/侧方 Collision Monitor 安全区和人工安全确认都齐全时才能 PASS。

阶段 4 算法和转换边界：

- Smac Hybrid-A* 负责生成满足 footprint 和最小转弯半径的 path。
- RPP 负责跟踪 path 并输出内部 `vx/wz`，不直接作为 STM32 控制接口。
- `twist_to_ackermann` 使用低速 bicycle model：`steering_angle_rad = atan(wheelbase_m * wz / vx)`。
- `vx≈0 && wz!=0` 属于阿克曼不可行原地旋转请求，adapter 必须停车或拒绝。
- adapter 必须对速度、转角、倒车、NaN/inf 和超时做限幅或停车处理，并通过 diagnostics 记录原因。

adapter 默认参数和判定：

| 参数 | 默认值 | 验收约束 |
| --- | --- | --- |
| `wheelbase_m` | `0.60` | 与阶段 1 车辆参数一致 |
| `max_steering_angle_rad` | `0.262` | 与阶段 1 转角限幅一致 |
| `max_auto_speed_mps` | `1.00` | 自动前进命令限幅；霍尔超速保护不是导航速度目标 |
| `max_reverse_speed_mps` | `0.60` | 仅阶段 4B 使用 |
| `min_turn_speed_mps` | `0.05` | 低于该 `abs(vx)` 不计算 `atan(wheelbase*wz/vx)` |
| `angular_deadband_radps` | `0.02` | 小于等于该 `abs(wz)` 视为零角速度 |
| `allow_reverse` | `false` | 阶段 4A 必须为 `false` |
| `input_timeout_sec` | `0.50` | 超时后 adapter 以停车输出维持安全 |
| `brake_on_stop` | `true` | 主动停车时 `brake=true` |
| `enable_on_command` | `true` | 正常命令输出时 `enable=true` |
| `publish_timeout_stop` | `true` | 输入超时时持续发布停车命令 |
| `diagnostics_rate_hz` | `10.0` | diagnostics 发布频率 |

| 输入/状态 | 输出动作 | diagnostics |
| --- | --- | --- |
| `vx=0` 且 `abs(wz)<=angular_deadband_radps` | 停车输出 | `stop_reason=zero_command` |
| `abs(vx)<min_turn_speed_mps` 且 `abs(wz)>angular_deadband_radps` | 停车输出，不计算转角 | `infeasible_spin=true`、`stop_reason=infeasible_spin` |
| NaN/inf | 停车输出 | `stop_reason=invalid_input` |
| 输入超时 | 持续停车输出直到收到新合法输入 | `stop_reason=input_timeout` |
| `vx<0` 且 `allow_reverse=false` | 停车输出 | `reverse=true`、`stop_reason=reverse_disabled` |
| 速度或转角超限 | 夹紧到允许值 | `speed_limited=true` 或 `steering_limited=true`，`stop_reason=none` |
| 正常可行输入 | 输出 Ackermann 命令 | `stop_reason=none` |

`stop_reason` 枚举固定为 `none`、`zero_command`、`invalid_input`、`input_timeout`、`infeasible_spin`、`reverse_disabled`。
adapter 停车输出固定为 `speed_mps=0.0`、`steering_angle_rad=0.0`、`enable=true`、`brake=brake_on_stop`、`clear_fault=false`、`emergency_stop=false`。
Collision Monitor 的障碍原因由 Collision Monitor 日志、`/safe_nav2_cmd_vel` 和 rosbag 证明，adapter 不凭零 Twist 猜测障碍原因。

Collision Monitor 默认安全区基线：

坐标以 `base_footprint` 为准，`x` 向前、`y` 向左；当前车身 footprint 为
`x=[-0.16, 0.7175]`、`y=[-0.285, 0.285]`。阶段 4A 必须至少实现前向两个 Collision Monitor 区。
ROS2 Humble 不使用后续版本才支持的 Collision Monitor `limit` action，前向绝对限速由 velocity smoother
和 `twist_to_ackermann` 承担：

| Zone | Polygon `[[x,y], ...]` | 动作 | 验收约束 |
| --- | --- | --- | --- |
| `front_stop` | `[[0.72,-0.35],[1.05,-0.35],[1.05,0.35],[0.72,0.35]]` | `stop` | 障碍进入该区时 `/safe_nav2_cmd_vel` 必须为停车或触发安全停车 |
| `front_slowdown` | `[[0.72,-0.45],[1.50,-0.45],[1.50,0.45],[0.72,0.45]]` | `slowdown` | 建议 slowdown ratio `0.35`；必须留下限速证据 |

阶段 4B 受控倒车必须额外实现后方/侧方区，否则 `reverse_plan_controlled` 只能记为 `未实现` 或 `BLOCKED`：

| Zone | Polygon `[[x,y], ...]` | 动作 | 验收约束 |
| --- | --- | --- | --- |
| `rear_stop` | `[[-0.55,-0.35],[-0.16,-0.35],[-0.16,0.35],[-0.55,0.35]]` | `stop` | 倒车障碍进入该区时停车 |
| `rear_slowdown` | `[[-1.20,-0.45],[-0.16,-0.45],[-0.16,0.45],[-1.20,0.45]]` | `slowdown` | 倒车限速证据齐全 |
| `left_side_stop` | `[[-0.16,0.285],[0.72,0.285],[0.72,0.45],[-0.16,0.45]]` | `stop` | 侧向贴近障碍时不允许自动倒车 PASS |
| `right_side_stop` | `[[-0.16,-0.45],[0.72,-0.45],[0.72,-0.285],[-0.16,-0.285]]` | `stop` | 侧向贴近障碍时不允许自动倒车 PASS |

必须检查：

- Nav2 lifecycle 节点处于 active。
- RViz 初始位姿设置后定位稳定。
- 目标点测试时 RPP 输出内部 `/nav2_cmd_vel`，Collision Monitor 输出 `/safe_nav2_cmd_vel`。

- 障碍场景能停车、限速、重规划或明确进入 `BLOCKED`。
- 失败时车辆停车或进入安全状态。

## 禁止用于验收的 legacy 入口

以下入口当前存在，但禁止作为阶段 1 至阶段 4 的正式验收入口：

```bash
ros2 launch turn_on_autoracer_robot turn_on_autoracer_robot.launch.py
ros2 launch autoracer_slam_toolbox slam.launch.py
ros2 launch autoracer_slam_toolbox slam.launch.py odom_frame:=odom_combined
ros2 topic pub /cmd_vel geometry_msgs/msg/Twist ...
```

禁止原因：

- `turn_on_autoracer_robot.launch.py` 使用 legacy/default 语义：`autoracer_robot` 订阅 `/cmd_vel`，发布 raw `/odom`。
- `autoracer_ekf.launch.py` 在 legacy/default 链路中输出 `/odom_combined`。
- 阶段验收要求 canonical `/odom`。
- 阶段验收要求正式底盘命令入口为 `/ackermann_cmd`。
- `slam.launch.py` 默认 `include_bringup:=true` 会拉起 legacy/default bringup。
- 阶段 4 中 `/cmd_vel` 只能作为 Nav2 内部中间控制量的隔离/remap 概念，不得直接作为底盘输入。

允许范围：

- 迁移对照。
- 手动调试。
- 旧链路兼容检查。

## 固定目标入口实现要求

以下入口是阶段固定目标入口。不得删除验收目标，不得把 legacy 入口改写为验收入口。

```bash
ros2 launch autoracer_bringup stage1_chassis.launch.py counts_per_meter:=<实测值> use_ekf:=true
ros2 launch autoracer_bringup stage2_path_tracking.launch.py counts_per_meter:=<实测值>

ros2 launch autoracer_bringup stage4_navigation.launch.py map:=/path/to/map.yaml counts_per_meter:=<实测值>
```

实现要求：

- `stage1_chassis.launch.py` 必须启动或替代当前 `ackermann_chassis.launch.py` 的底盘桥。
- `stage1_chassis.launch.py` 必须补齐 IMU、URDF/TF 和可选 RViz。
- `stage2_path_tracking.launch.py` 必须复用阶段 1 底盘链路。
- `stage3_mapping.launch.py` 必须使用 phase-1 canonical `/odom`。
- `stage4_navigation.launch.py` 必须使用已保存地图、LiDAR、AMCL、Smac Hybrid-A*、Regulated Pure Pursuit、Collision Monitor、`twist_to_ackermann` 和 `/ackermann_cmd` 输出链路。
