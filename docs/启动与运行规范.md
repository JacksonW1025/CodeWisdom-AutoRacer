根目录文档只维护系统约束、阶段目标、接口协议和验证边界。

阶段 1-4 的验收契约、追踪表、PASS/FAIL 规则和人工边界以根目录
`../../docs/阶段路线图.md` 和 `../../docs/开发流程与验证规范.md` 为准。本文只记录上位机可执行入口和 topic/launch 检查；测试结果必须回填根目录阶段验收追踪表。

### 参数表

`ackermann_chassis.launch.py` 参数：

| 参数 | 当前默认值 | 验收约束 |
| --- | --- | --- |
| `usart_port_name` | `/dev/ttyACM0` | 按车载 STM32 串口设备设置 |
| `serial_baud_rate` | `115200` | 必须与 STM32 UART4 一致 |
| `counts_per_meter` | `0.0` | `/wheel_odom` 验收必须大于 `0` |
| `use_ekf` | `false` | `/odom` 验收必须为 `true` |

`ackermann_chassis_bridge` 配置文件参数：

| 参数 | 当前值 |
| --- | --- |
| `odom_frame_id` | `odom` |
| `robot_frame_id` | `base_footprint` |
| `wheelbase_m` | `0.60` |
| `max_steering_angle_rad` | `0.262` |
| `min_turn_speed_mps` | `0.05` |
| `max_auto_speed_mps` | `1.00` |
| `max_reverse_speed_mps` | `0.60` |
| `publish_power_voltage` | `true` |
| `serial_poll_period_ms` | `5` |
| `serial_reconnect_period_ms` | `1000` |

这些速度限幅对应 STM32 固件默认值 `APP_ORIN_VX_FORWARD_CAP_MMPS=1000` 和
`APP_ORIN_VX_REVERSE_CAP_MMPS=600`。`APP_ORIN_VX_MAX_DEFAULT_MMPS=500` 是旧变量的回退值；
当前显式前进/倒车 cap 非零时不作为有效自动命令上限。
`APP_HALL_SPEED_LIMIT_MMPS=5000` / `APP_HALL_SPEED_LIMIT_RELEASE_MMPS=4500`
是霍尔反馈超速保护阈值，不是自动控制命令限速，也不是阶段 1/2 测试速度目标。

### 启动前检查

```bash
ros2 pkg prefix turn_on_autoracer_robot
ros2 pkg executables turn_on_autoracer_robot
test -e /dev/ttyACM0
python3 tools/acceptance/stage1_acceptance_check.py --mode offline
```

必须确认：

- STM32 固件已烧录。
- UART4 线缆、波特率、共地和电平已确认。
- RC 接管、急停、刹车和断电路径可用。
- `counts_per_meter` 有实测值，或本次只做串口/telemetry bringup。
- `/imu/data_raw` 已由 `hipnuc_imu` 发布，或本次不验收 `/odom`。
- `/imu/data` 已由 `imu_filter_madgwick` 发布，或本次不验收 `/odom`。

`counts_per_meter` 标定：

```text
counts_per_meter = abs(sum(hall_delta_count)) / measured_distance_m
```

记录要求：

- 单次标定距离不小于 `2.0 m`。
- 同一方向至少执行 `3` 次。
- 保存每次 `measured_distance_m`、`sum(hall_delta_count)`、单次结果和均值。
- 最大值和最小值相对均值偏差不超过 `10%`。
- 阶段 1 当前采用 `counts_per_meter=13.545`，来源为霍尔 `10` 计数/圈和轮径 `0.235 m`，并经三次大于 `2 m` 的直线采集确认。
- 无标定记录时不记录 `/wheel_odom` 或 `/odom` PASS。

### 启动后检查

```bash
ros2 node list
ros2 topic list
ros2 topic echo /chassis_state --once
ros2 topic hz /chassis_state
ros2 topic hz /wheel_odom
ros2 topic hz /odom
ros2 run tf2_ros tf2_echo odom base_footprint
python3 tools/acceptance/stage1_acceptance_check.py --mode live
python3 tools/acceptance/stage1_acceptance_check.py --mode live --require-odom
```

必须检查：

- `/chassis_state` 有数据。
- `/wheel_odom` 在 `counts_per_meter>0` 且 telemetry 有效时发布。
- `/odom` 在 `use_ekf:=true` 且 `/imu/data`、`/wheel_odom` 有效时发布。
- 通信超时后 STM32 进入停车状态。
- 非零运动命令测试必须满足架空或安全场地条件。

验收工具边界：

- `stage1_acceptance_check.py --mode offline` 只检查协议帧、telemetry 解析、`counts_per_meter` 门禁、launch 契约和串口异常恢复契约，不打开串口、不发布运动命令。
- `stage1_acceptance_check.py --mode live` 只检查已运行 ROS topic 和类型，不发布运动命令。
- `--require-odom` 用于 `/imu/data`、`/wheel_odom` 和 EKF `/odom` 均预期存在的检查场景。
- 工具输出必须回填测试记录；失败项不得人工改成 PASS。

标准非零运动命令只用于架空或低速安全场地：

```bash
timeout 2s ros2 topic pub --rate 10 /ackermann_cmd autoracer_interfaces/msg/AckermannChassisCommand "{header: {frame_id: ''}, speed_mps: 0.10, steering_angle_rad: 0.0, enable: true, brake: false, clear_fault: false, emergency_stop: false}"
timeout 2s ros2 topic pub --rate 10 /ackermann_cmd autoracer_interfaces/msg/AckermannChassisCommand "{header: {frame_id: ''}, speed_mps: 0.10, steering_angle_rad: 0.15, enable: true, brake: false, clear_fault: false, emergency_stop: false}"
timeout 2s ros2 topic pub --rate 10 /ackermann_cmd autoracer_interfaces/msg/AckermannChassisCommand "{header: {frame_id: ''}, speed_mps: 0.10, steering_angle_rad: -0.15, enable: true, brake: false, clear_fault: false, emergency_stop: false}"
ros2 topic pub --once /ackermann_cmd autoracer_interfaces/msg/AckermannChassisCommand "{header: {frame_id: ''}, speed_mps: 0.0, steering_angle_rad: 0.0, enable: true, brake: true, clear_fault: false, emergency_stop: false}"
ros2 topic pub --once /ackermann_cmd autoracer_interfaces/msg/AckermannChassisCommand "{header: {frame_id: ''}, speed_mps: 0.0, steering_angle_rad: 0.0, enable: false, brake: true, clear_fault: false, emergency_stop: true}"
```

记录要求：

- `straight_low_speed`：直行命令，`actual_speed_mps` 和 `/wheel_odom` 方向正确。
- `left_low_speed`：左转命令，`steering_angle_rad` 为正。
- `right_low_speed`：右转命令，`steering_angle_rad` 为负。
- `stop_command`：停车命令，速度回零且 `brake_active` 证据完整。
- `timeout_stop`：停止发布命令后记录 `command_timeout` 和速度回零。
- `estop_command`：上位机急停命令后记录 `estop_active` 和速度回零。

## 阶段 2：路径跟踪

### 当前状态

阶段 2 固定目标入口、test path publisher、point-goal path publisher、Pure Pursuit tracker 和 diagnostics publisher 已实现。

阶段 2 验收 case 固定为 `straight_2m`、`left_arc_r2m`、`right_arc_r2m`、`s_curve`、`stop_at_end`，并扩展 `goal_2m_2m` 用于验证无避障点目标跟踪。未实现对应 fixture、tracker、诊断或上机证据时，测试记录必须标为 `未实现`、`FAIL` 或 `BLOCKED`，不得记为通过。

阶段 2 实现归属：

- 新建 `autoracer_path_tracking` 包承载 test path publisher、Pure Pursuit tracker；tracker 同步发布 diagnostics。
- `autoracer_path_tracking/goal_path_publisher` 负责把当前 `/odom` 位姿到 `goal_x_m/goal_y_m` 的直线生成为 `/path_tracking/path`。
- 新建 `autoracer_bringup/stage2_path_tracking.launch.py` 作为固定目标入口。
- 新建 `autoracer_bringup/stage2_goal_tracking.launch.py` 作为 odom 点目标入口。
- `stage2_path_tracking.launch.py` include 阶段 1 Ackermann 底盘链路，并启动 `autoracer_path_tracking` 中的阶段 2 节点。
- 阶段 2 tracker 默认启用零速预使能门控：发出非零运动命令前，先持续发布 `speed_mps=0`、`enable=true`，直到 `/chassis_state` 显示 `auto_enabled=true`、`command_timeout=false`、`rc_override_active=false`、`estop_active=false`。
- tracker 算法固定为 Pure Pursuit。

当前已准备阶段 2 path fixture 和 fake-odom tracker 检查：

```bash
python3 tools/acceptance/stage2_path_fixture_check.py
python3 tools/acceptance/stage2_tracker_fake_odom_check.py
```

fixture 位置：

```text
tools/acceptance/fixtures/stage2_paths/
```

这些 fixture 是 `nav_msgs/Path` 语义的标准输入，用于 tracker 离线和实车验收。fixture 静态检查本身不发布
`/ackermann_cmd`，fake-odom 检查只证明 tracker 软件输出满足方向、限幅和到点停车契约，不代表实车阶段 2 PASS。

固定目标入口：

```bash
ros2 launch autoracer_bringup stage2_path_tracking.launch.py counts_per_meter:=<实测值>
```

该命令当前可运行；实车运行前必须满足阶段 1 安全链路、`counts_per_meter` 标定和低速可接管条件。
启动后 tracker 会先执行零速预使能。若底盘停留在空闲 RC passthrough 或 command timeout 状态，
tracker 必须保持零速，不得直接释放非零路径命令。

点目标入口示例，不带避障，只验证从当前 `/odom` 位姿到目标点的路径跟踪：


现场明确批准更高速度时，可以同时覆盖 tracker 的速度上限，例如 `target_speed_mps:=1.00 max_target_speed_mps:=1.00`。

| `max_target_speed_mps` | 默认 `0.25`；只在现场明确批准的实车测试中覆盖，最高不得超过 `1.00` |
| `wheelbase_m` | 必须与阶段 1 车辆参数一致 |
| `max_steering_angle_rad` | 必须与阶段 1 车辆参数一致 |
| `lookahead_m` | 默认 `0.60`，必须写入测试记录 |
| `goal_tolerance_m` | 默认 `0.05`；控制停车窗口。`straight_2m` 实车验收按 `0.10 m` 窗口，要求前进距离不低于 `1.90 m` |
| `control_rate_hz` | 默认 `20` |
| `allow_reverse` | 固定 `false` |
| `prearm_zero_before_motion` | 阶段 2 launch 默认 `true`；非零运动前零速预使能 |
| `prearm_timeout_s` | 默认 `3.0`；超过后只告警，仍保持零速直到底盘自动控制就绪 |

阶段 2 tracker 的 `0.25 m/s` 是默认低速软件上限；经现场批准覆盖到更高速度时，
仍受 STM32 前进命令上限 `1.00 m/s` 约束。霍尔超速保护 `5.0/4.5 m/s`
只用于异常保护，不得写作阶段 2 目标速度。

## 禁止用于验收的 legacy 入口

以下入口当前存在，但禁止作为阶段 1 至阶段 4 的正式验收入口：

```bash
ros2 launch turn_on_autoracer_robot turn_on_autoracer_robot.launch.py
ros2 launch autoracer_slam_toolbox slam.launch.py
ros2 launch autoracer_slam_toolbox slam.launch.py odom_frame:=odom_combined
ros2 topic pub /cmd_vel geometry_msgs/msg/Twist ...
```

禁止原因：

- `turn_on_autoracer_robot.launch.py` 使用 legacy/default 语义：`autoracer_robot` 订阅 `/cmd_vel`，发布 raw `/odom`。
- `autoracer_ekf.launch.py` 在 legacy/default 链路中输出 `/odom_combined`。
- 阶段验收要求 canonical `/odom`。
- 阶段验收要求正式底盘命令入口为 `/ackermann_cmd`。
- `slam.launch.py` 默认 `include_bringup:=true` 会拉起 legacy/default bringup。
- 阶段 4 中 `/cmd_vel` 只能作为 Nav2 内部中间控制量的隔离/remap 概念，不得直接作为底盘输入。

允许范围：

- 迁移对照。
- 手动调试。
- 旧链路兼容检查。
